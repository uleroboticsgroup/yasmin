Orthogonal Regions Demo (Python)

This tutorial demonstrates how to run multiple independent state machines in parallel using YASMIN's OrthogonalState container. Orthogonal regions allow you to define separate, independently executing state machines (regions) that run concurrently within a single parent state. Each region has its own internal state and transitions, while the OrthogonalState coordinates their execution and maps their combined outcomes to the next transition.

Background

The OrthogonalState container enables truly independent parallel execution of multiple state machines (regions). Unlike Concurrence which runs individual states in parallel, orthogonal regions run full state machines in parallel, each with its own lifecycle, blackboard, and outcome resolution. This is useful when you need to:

  • Model independent behaviors that have their own complex state logic
  • Run concurrent missions where each region has multiple sequential steps
  • Combine independent subsystems (navigation, manipulation, perception) into a single coordinated behavior
  • Synchronize regions at specific points using JoinState

1. Create the WorkerState

This state counts iterations and returns "working" until the maximum count is reached, then returns "done". It accepts a name for logging, a maximum iteration count, and an optional sleep duration. This simple looping state is used by both regions to demonstrate independent progress. 

class WorkerState(State):
    def __init__(self, name: str, max_count: int, sleep_ms: int = 500) -> None:
        super().__init__(["working", "done"])
        self.name = name
        self.counter = 0
        self.max_count = max_count
        self.sleep_ms = sleep_ms

    def execute(self, blackboard: Blackboard) -> str:
        yasmin.YASMIN_LOG_INFO(
            f"WorkerState [{self.name}]: iteration {self.counter + 1}/{self.max_count}"
        )
        time.sleep(self.sleep_ms / 1000.0)

        self.counter += 1
        if self.counter >= self.max_count:
            return "done"
        return "working"

2. Create Regions

Each region is a full StateMachine with its own internal states and transitions. Region A runs for 3 iterations, Region B runs for 5 iterations. Because they execute in parallel, the total time is determined by the slowest region (B). Each region has a single outcome "done" that it emits when its internal workflow completes. 

def main() -> None:
    rclpy.init()
    set_ros_loggers()
    yasmin.YASMIN_LOG_INFO("yasmin_orthogonal_demo")

    # Region A: 3 iterations
    reg_a = StateMachine(outcomes=["done"])
    reg_a.set_name("Region A")
    reg_a.add_state(
        "WORK",
        WorkerState("A", 3, 300),
        transitions={"working": "WORK"},
    )
    reg_a.set_start_state("WORK")

    # Region B: 5 iterations
    reg_b = StateMachine(outcomes=["done"])
    reg_b.set_name("Region B")
    reg_b.add_state(
        "WORK",
        WorkerState("B", 5, 300),
        transitions={"working": "WORK"},
    )
    reg_b.set_start_state("WORK")

3. Create the OrthogonalState Container

The OrthogonalState takes two key parameters: a default outcome (returned when no outcome map matches) and an outcome map that defines how region outcomes map to the container's outcome. Here, the outcome map specifies that "success" is returned when both Region A and Region B emit "done". If either region finishes with an unmatched outcome, the default "timeout" is returned. 

    # Orthogonal state runs both regions in parallel
    ort = OrthogonalState(
        default_outcome="timeout",
        outcome_map={"success": {"A": "done", "B": "done"}},
    )
    ort.set_description(
        "Runs Region A and Region B in parallel and maps their combined outcomes."
    )
    ort.add_region("A", reg_a)
    ort.add_region("B", reg_b)

4. Create the Root State Machine

Add the OrthogonalState to a root state machine and define transitions from its outcomes. When the orthogonal container returns "success" (both regions done), the root SM transitions to its "success" outcome. When "timeout" is returned, it transitions to the "timeout" outcome. 

    # Root state machine
    sm = StateMachine(outcomes=["success", "timeout"], handle_sigint=True)
    sm.set_description(
        "Demonstrates orthogonal state execution with two parallel regions."
    )
    sm.add_state(
        "PARALLEL",
        ort,
        transitions={"success": "success", "timeout": "timeout"},
    )
    sm.set_start_state("PARALLEL")

    YasminViewerPub(sm, "YASMIN_ORTHOGONAL_DEMO")

    try:
        outcome = sm()
        yasmin.YASMIN_LOG_INFO(outcome)
    except Exception as e:
        yasmin.YASMIN_LOG_WARN(e)

    if rclpy.ok():
        rclpy.shutdown()


if __name__ == "__main__":
    main()

5. Run the Demo

Execute the orthogonal regions demonstration to observe parallel state machine execution across independent regions. 

ros2 run yasmin_demos orthogonal_demo.py

How It Works

The OrthogonalState manages region lifecycles:

  • All regions start simultaneously when the orthogonal state is entered
  • Each region runs its own internal state machine independently
  • Regions share no internal state — they are truly isolated
  • The orthogonal state waits until all regions complete (or a timeout occurs)
  • The outcome map combines region outcomes into a single result
