Callback Signal

CallbackSignal is a native YASMIN blackboard object for cross-language coordination between Python and C++ states. It allows states to register callbacks and later trigger all registered callbacks from either language.

Key Features:

  • Cross-language callbacks: Store Python and C++ callbacks in the same signal instance
  • Thread-safe triggering: Supports synchronous and asynchronous execution
  • Blackboard integration: Share a signal through the blackboard like any other supported object
  • Event-style coordination: Useful for cancellation, cleanup, reset, and other cooperative control flows

Overview

Before CallbackSignal, the blackboard was mainly used to exchange data. The new signal object extends this model so states can also exchange behavior. A state can create a signal, place it in the blackboard, and other states can register callbacks on it. When the signal is triggered, all currently registered callbacks are executed.

This is especially useful when mixing Python and C++ states in the same state machine through the YASMIN Factory. It complements primitive blackboard values and can be combined with ROS Serialization States when both event-style coordination and ROS interface exchange are needed.

API

Function Description
add_callback(callback) Registers a callback and returns its callback id.
remove_callback(callback_id) Removes a previously registered callback by id.
trigger() Executes all registered callbacks synchronously and blocks until they finish.
trigger_async() Executes all registered callbacks asynchronously and returns a future-like handle.

Callbacks do not take arguments and do not return values. This keeps the interface simple and consistent across Python and C++.

Execution Semantics

CallbackSignal is thread-safe. Triggering uses snapshot semantics, which means the current callback list is copied before execution starts. If another thread adds or removes callbacks while a trigger is already running, the running trigger continues with its own snapshot and the modification only affects future triggers.

trigger() propagates callback failures directly. The object returned by trigger_async() allows callers to wait for completion, check whether an exception occurred, and inspect the stored exception message.

Python Example

The following example creates a shared signal in the blackboard and uses it for cooperative cancellation. 

from yasmin import Blackboard, CallbackSignal

blackboard = Blackboard()
blackboard["cancel_signal"] = CallbackSignal()


def cancel_worker() -> None:
    print("worker got cancel request")


def cancel_monitor() -> None:
    print("monitor got cancel request")


signal = blackboard["cancel_signal"]
worker_cb = signal.add_callback(cancel_worker)
monitor_cb = signal.add_callback(cancel_monitor)

signal.trigger()

signal.remove_callback(worker_cb)
future = signal.trigger_async()
future.wait()

C++ Example

The same signal can also be created and used from C++. In mixed Python and C++ state machines, both languages can register callbacks on the same object. 

#include <iostream>
#include <memory>

#include "yasmin/blackboard.hpp"
#include "yasmin/callback_signal.hpp"

int main() {
  auto blackboard = std::make_shared<yasmin::Blackboard>();
  auto signal = std::make_shared<yasmin::CallbackSignal>();

  blackboard->set<std::shared_ptr<yasmin::CallbackSignal>>(
      "cancel_signal", signal);

  const std::size_t callback_id = signal->add_callback([]() {
    std::cout << "cancel callback from C++" << std::endl;
  });

  signal->trigger();

  signal->remove_callback(callback_id);

  auto future = signal->trigger_async();
  future.wait();

  if (future.has_exception()) {
    std::cerr << future.get_exception_message() << std::endl;
  }

  return 0;
}

Typical Use Cases

  • Cancel another state running in a concurrence
  • Trigger cleanup or shutdown hooks
  • Notify multiple states that a branch has finished
  • Broadcast reset or recovery requests

CallbackSignal is designed for simple event-style actions. It is not intended to replace structured blackboard data or message passing.
